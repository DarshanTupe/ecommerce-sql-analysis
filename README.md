# Ecommerce SQL Analysis

## Project Overview

In this project, I worked with an ecommerce dataset and used SQL in PostgreSQL to analyze customer orders, revenue trends, and product performance.

The goal of this project was mainly to practice SQL and try to answer some common business questions using data.

The dataset contains information about customers, orders, payments, and products, which makes it useful for exploring different types of analysis like revenue trends, customer behavior, and delivery performance.

---

## Tables Used

The dataset includes the following tables:

| Table | Description |
|------|-------------|
| customers | customer information and location |
| orders | order details and timestamps |
| payments | payment information for each order |
| order_items | items included in orders |
| products | product details and categories |

---

## Some Questions I Explored

While working on this dataset I tried to answer questions like:

- What is the total revenue generated by the business?
- How many orders and customers are there?
- What are the top months with the highest number of orders?
- What is the average order value?
- How much money does each customer spend?
- What percentage of customers are repeat buyers?
- How long does delivery usually take?
- How many orders were delivered late?
- Which products sold the most units?
- Which states generate the most revenue?

## SQL Concepts I Practiced

In this project I used several SQL concepts such as:

- JOINs
- Aggregations (SUM, COUNT, AVG)
- CASE statements
- Date functions
- Common Table Expressions (CTEs)
- Window functions like RANK and LAG
- Grouping and filtering
